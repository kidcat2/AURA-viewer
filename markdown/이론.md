## 1. 가우시안

### 요소

1. positions — 각 가우시안의 3D 중심 좌표 (x, y, z)
2. normals — 표면 법선 벡터 (nx, ny, nz). 3DGS 렌더엔 거의 안 쓰임
3. colors — 기본 색(SH 0차, f_dc). RGB로 쓰려면 `C0·f_dc + 0.5` 복원 필요
4. sh — 시점 의존 색을 위한 고차 구면조화 계수 (f_rest_0~44)
5. opacities — 불투명도. inverse-sigmoid로 저장됨 → `sigmoid`로 복원
6. scales — 타원체 세 축의 크기. log로 저장됨 → `exp`로 복원
7. rotations — 타원체의 방향(쿼터니언, rot_0~3). normalize 후 회전행렬로 변환

### 깊이 정렬

1. 각 가우시안은 불투명도 `opacity`를 갖고 있음.
2. GL, DX에서의 렌더링은 다음과 같이 진행됨
    - 불투명한 물체 : depth buffer를 이용. 현재 픽셀에 색을 결정할 때, 현재 물체의 depth와 이미 저장된 depth값을 비교해서 가까울 때만 덧칠함.
    - 반투명한 물체 : 모든 물체를 깊이를 기준으로 정렬한 뒤, 가장 멀리 있는 물체부터 알파 블렌딩으로 색상을 결정함.
3. 따라서 가우시안도 깊이값에 따라 미리 정렬을 해줘야 함.

### 매우 작은 가우시안

1. 매우 작은 가우시안은 픽셀과 픽셀 사이에 존재할 수 있음.
2. 이러한 가우시안은 카메라 이동에 픽셀에 들어왔다-나갔다 를 반복하게 되고, 렌더링이 됐다-안됐다가 반복되면서 깜빡이는 현상(flickering)이 발생


## 2. 렌더링

### CSS 픽셀과 실제 픽셀
- 실제 픽셀 : 화면 하드웨어에 박힌 진짜 픽셀. =모니터의 해상도
- CSS 픽셀 : 브라우저가 레이아웃에 쓰는 논리적 단위. `width: 100px` 이 이 기준임.
- DPR(Device Pixel Ratio) : 실제 픽셀 / CSS 픽셀

### 실제 렌더링 화면 크기 정하기
1. React는 div를 받음. Three.js의 Canvas는 div안에 속한 객체임. div의 크기는 `<div width=..>` 로 사용자가 정함.
2. Three.js Canvas 는 div에서 다시 일부 크기를 잡아서 canvas의 크기로 써야함. 그렇게 하지 않으면 Three.js의 기본 Canvas 사이즈인 300x150px로 생성
3. Three.js의 Canvas는 CSS 픽셀을 `setSize`로 잡았지만 실제 화면에는 나중에 DPR만큼 확대되어서 그려짐. 
4. 따라서 DPR만큼 미리 확대해서 그리기 위해 사용하는 것이 `setPixelRatio`
