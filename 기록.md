## 가우시안 렌더링 기록

### 저장 포맷과 복원
1. f_dc
- 문제: `f_dc`를 RGB로 변환 없이 raw로 사용, f_dc는 sh 계수이므로 복원 필요 → 값이 1.0 초과로 흰색 클램프
- 해결: SH 0차 변환 `vColor = C0 * f_dc + 0.5` (C0=0.282…) 적용 후 clamp

2. opacity
- 문제 : `opacity` 는 inverse sigmoid 로 저장됨.
- 해결 : `opacity = opacity 1.0 / (1.0 + exp(-opacity))` 적용

### 렌더링
1. SH 45개를 attribute로 전송 불가
   - 문제: WebGL attribute는 최대 vec4 + 총 16슬롯 → 45개(vec3 15개)는 한계 초과
   - 해결: 첫 렌더엔 SH 생략, 추후 DataTexture(텍스처 샘플링) 방식으로 전송 예정

2. 블렌딩 단계에서 다시 흰색/누적 포화
   - 문제: material에 `transparent` 미설정으로 블렌딩 꺼짐 + RGB까지 `g`를 곱해 색 왜곡
   - 해결: frag를 `vec4(vColor, vOpacity * g)`로, material에 `transparent/depthWrite:false/depthTest:false/NormalBlending` 설정

3. 정렬 부재로 화면이 뿌옇고(hazy) 디테일이 뭉개짐
   - 문제: 깊이 정렬 부재로 splat이 카메라 거리와 무관한 임의 순서로 합성됨 → 알파 over 연산은 순서 의존이라 결과가 흐려짐
   - 해결: 각 splat을 modelView로 뷰 공간 변환 → z로 깊이 계산 → index 배열을 뒤→앞(작은 z부터) 정렬 → `geometry.setIndex(order)`로 그리기 순서 변경 (데이터는 고정, index만 정렬)

4. depthWrite를 켜니 splat이 드문드문 빠져 보임
   - 문제: splat 사각형의 거의 투명한 가장자리까지 깊이를 기록 → 뒤쪽 splat이 깊이 테스트에서 탈락 (정렬도 없어 무작위로 살아남음)
   - 해결: `depthWrite:false`로 모든 splat이 탈락 없이 블렌딩에 참여하도록 유지
